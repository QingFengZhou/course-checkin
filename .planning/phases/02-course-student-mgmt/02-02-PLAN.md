  <behavior>
    - POST /api/courses: requires auth, validates body with createCourseSchema, sets teacher_id from JWT session, inserts into courses table, returns { id, name, semester }
    - GET /api/courses: requires auth, returns array of courses with student counts for the authenticated teacher only (WHERE teacher_id = session.user.id)
    - Both return 401 if not authenticated
    - Error format: { error: "message" } matching Phase 1 convention
  </behavior>
  <action>
    Create `src/app/api/courses/route.ts` following the pattern from `src/app/api/auth/login/route.ts`:

    **POST handler:**
    - Call `getAuthSession(request)` — reject with 401 if not authenticated
    - Parse request body with `createCourseSchema.safeParse(body)` — reject with 400 + first error message if invalid
    - Insert into courses table: `db.insert(courses).values({ name, semester, teacherId: session.user.id }).returning()`
    - Return 201 with `{ data: { id, name, semester } }`

    **GET handler:**
    - Call `getAuthSession(request)` — reject with 401 if not authenticated
    - Query courses with student count using Drizzle `groupBy`: `db.select({ id: courses.id, name: courses.name, semester: courses.semester, teacherId: courses.teacherId, createdAt: courses.createdAt, updatedAt: courses.updatedAt, studentCount: count(students.id) }).from(courses).leftJoin(students, eq(courses.id, students.courseId)).where(eq(courses.teacherId, session.user.id)).groupBy(courses.id)`
    - If Drizzle groupBy syntax is unclear, fallback: query courses first, then for each course do a `count(*)` subquery or use a raw SQL fragment
    - Return 200 with `{ data: courses }` where each course has a `studentCount` field (per D-2.01 cards show student count)

    Import from: `@/lib/auth` (getAuthSession), `next/server` (NextRequest, NextResponse), `@/db` (db), `@/db/schema` (courses, students), `drizzle-orm` (eq, count), `@/lib/zod-schemas` (createCourseSchema)
  </action>

  <name>Task 2: Create course delete API with ownership check</name>
  <files>src/app/api/courses/[id]/route.ts</files>
  <behavior>
    - DELETE /api/courses/[id]: requires auth, verifies course belongs to teacher, deletes course (cascade deletes students per D-2.03)
    - Returns 404 if course not found or not owned by teacher
    - Returns 200 with { message: "Course deleted" } on success
  </behavior>
  <action>
    Create `src/app/api/courses/[id]/route.ts`:

    **DELETE handler:**
    - Call `getAuthSession(request)` — reject with 401 if not authenticated
    - Extract `id` from route params: `const { id } = await params`
    - Verify ownership: `const result = await db.select().from(courses).where(and(eq(courses.id, id), eq(courses.teacherId, session.user.id))).limit(1)`
    - If result.length === 0: return 404 with `{ error: "Course not found or you don't have permission" }`
    - Delete: `await db.delete(courses).where(eq(courses.id, id))` — cascade deletes students automatically (per D-2.03)
    - Return 200 with `{ message: "Course deleted" }`

    Import from: `@/lib/auth` (getAuthSession), `next/server` (NextRequest, NextResponse), `@/db` (db), `@/db/schema` (courses), `drizzle-orm` (eq, and)
  </action>
  <verify>
    <automated>curl -s -X DELETE http://localhost:3000/api/courses/nonexistent-id -H "Cookie: cc_session=VALID_TOKEN" | grep -c "not found"</automated>
  </verify>
  <done>DELETE /api/courses/[id] verifies ownership before deleting, returns 404 for non-owned courses, cascade deletes enrolled students via DB constraint.</done>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API | Untrusted request bodies, course IDs, student IDs cross this boundary |
| API → DB | Server-side only, but teacher_id must come from session not request |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Elevation of Privilege | All course/student API routes | mitigate | `getAuthSession()` at entry of every handler, 401 on missing/invalid session |
| T-02-07 | Information Disclosure | GET /api/courses, GET /api/courses/[id]/students | mitigate | WHERE clause filters by `teacher_id = session.user.id` — no teacher can see another's data |
| T-02-08 | Tampering | POST /api/courses | mitigate | `teacherId` set from JWT `session.user.id`, never from request body. Zod validates name/semester only |
| T-02-09 | Tampering | DELETE /api/courses/[id] | mitigate | Ownership check: `and(eq(courses.id, id), eq(courses.teacherId, session.user.id))` before delete |
| T-02-10 | Elevation of Privilege | POST/DELETE /api/courses/[id]/students | mitigate | `verifyCourseOwnership()` called before any student mutation; returns 404 for non-owners |
| T-02-11 | Repudiation | All mutations | accept | Phase 2 has no audit log requirement; audit trail will be added in Phase 5 (HIST-01) |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed — uses existing drizzle-orm, zod, jose |
</threat_model>
