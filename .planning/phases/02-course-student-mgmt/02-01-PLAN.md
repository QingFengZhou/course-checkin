<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API → DB | All DB writes go through API routes (server-side only, no client DB access) |
| Client → API | Request bodies must be validated with Zod before processing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | Course/student API routes | mitigate | Every API route calls `getAuthSession()` and rejects non-authenticated requests with 401 |
| T-02-02 | Information Disclosure | GET /api/courses | mitigate | Query filters by `teacher_id = session.user.id` — teachers only see their own courses |
| T-02-03 | Tampering | Course/student creation | mitigate | Zod validation on all request bodies; teacher_id derived from JWT session, not request body |
| T-02-04 | Tampering | Course deletion by non-owner | mitigate | DELETE route verifies `course.teacher_id === session.user.id` before deleting (per D-2.03 cascade) |
| T-02-05 | Elevation of Privilege | Student enrollment to another teacher's course | mitigate | POST /api/courses/[id]/students verifies course ownership before enrolling |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan — only schema and type files created |
</threat_model>
