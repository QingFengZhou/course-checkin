<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → API | Client-side fetches send auth cookie; API validates server-side |
| UI state → API | UI assumes success before API confirms; race conditions possible |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-12 | Information Disclosure | Dashboard UI | mitigate | All data fetched from APIs that already enforce ownership (T-02-07). UI displays only what API returns |
| T-02-13 | Tampering | Client-side fetch payloads | mitigate | Server-side Zod validation (T-02-03, T-02-10) catches any manipulated payloads. UI validation is UX-only |
| T-02-14 | Session Hijacking | Auth cookie in fetch | mitigate | `credentials: 'include'` sends HTTP-only cookie; server validates JWT signature on every request |
| T-02-15 | UX Spoofing | Delete confirm dialogs | mitigate | `window.confirm` with clear warning message about cascade delete ("这将同时删除该课程下的所有学生") per D-2.03 |
| T-02-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed — uses existing Next.js, Tailwind, React |
</threat_model>
