# 04-01: WebSocket Server Setup

## Objective

Create a custom server entry point that runs both HTTP (Next.js request handler) and WebSocket on the same port, with connection lifecycle management and heartbeat.

## Tasks

### Task 1: Create server.ts

<read_first>
- server.ts (NEW — project root)
- package.json
- next.config.ts or next.config.js (if exists)
- src/app/layout.tsx (root layout)
</read_first>

<action>
Create `server.ts` in project root:

1. Import `http` from `node:http` and `{ createServer }` from `ws`
2. In dev mode, import Next.js dev request handler; in production, import the built handler
3. Create `http.Server`, attach `ws.Server` with `{ noServer: true }` (not directly on HTTP server, handle upgrade manually)
4. Handle `server.on('upgrade', ...)` — verify path matches `/_next/webpack-hmr` (let Next.js handle dev HMR) or WebSocket upgrade; pass unmatched upgrades to Next.js
5. Add `heartbeat` interval — ping every 30s, close connections that miss 3 pongs
6. Listen on `process.env.PORT || 3000`
7. Handle graceful shutdown: `SIGTERM`/`SIGINT` → close WS server → close HTTP server → exit
</action>

<acceptance_criteria>
- `server.ts` exists in project root
- Running `npx tsx server.ts` starts HTTP server on port 3000
- `curl http://localhost:3000/login` returns 200 (serves pages)
- WebSocket connects successfully via `ws://localhost:3000` (test with node --eval)
- Heartbeat interval is active (30s ping, close on 3 missed pongs)
- SIGTERM/SIGINT stops the server gracefully without hanging
</acceptance_criteria>

```typescript
// Message types for WS protocol (D-4.04, D-4.05, D-4.06)

export interface WsMessage {
  type: string;
  sessionId: string;
  payload: unknown;
}

// Server → Client
export interface AttendanceUpdatePayload {
  checkedInCount: number;
  totalStudents: number;
  newCheckIn: {
    studentId: string;
    studentName: string;
  } | null;
}

export interface SessionEndedPayload {
  reason: "timeout" | "teacher_ended";
}

// Client → Server
export interface SubscribePayload {
  sessionId: string;
}

// Type constants
export const WS_EVENTS = {
  ATTENDANCE_UPDATE: "attendance_update",
  SESSION_ENDED: "session_ended",
  SESSION_ERROR: "session_error",
  SUBSCRIBE: "subscribe",
  UNSUBSCRIBE: "unsubscribe",
  PING: "ping",
  PONG: "pong",
} as const;
```
</action>
