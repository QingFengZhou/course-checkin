# 04-04: Real-Time Dashboard UI

## Objective

Enhance the teacher's check-in page with a rich real-time dashboard: large attendance rate display, live progress bar, scrollable present/absent student lists. Also add real-time status indicators to the dashboard course cards.

## Tasks

### Task 1: Upgrade check-in page status panel

<read_first>
- src/app/(protected)/checkin/[courseId]/components/CheckInPageClient.tsx
- src/lib/use-checkin-session.ts (from 04-03)
- src/lib/ws-types.ts
</read_first>

<action>
Replace the existing simple status panel (line 320-332) with a rich dashboard:

1. **Attendance Rate Display:**
   - Large percentage number (e.g., "67%")
   - Color-coded: green (≥80%), yellow (≥50%), red (<50%)
   - Progress bar below the percentage

2. **Live Count:**
   - "已签到 X / Y 人" with larger text
   - Animated counter when number changes (CSS transition on number)

3. **Student Lists (side by side or tabs for mobile):**
   - **已签到列表** (left/green): scrollable list of students who have checked in
   - **未签到列表** (right/gray): scrollable list of students not yet checked in
   - Each item shows: student ID, student name
   - Lists are scrollable with max-height, virtualized if > 50 students per list
   - Use CSS `overflow-y: auto` with fixed row height (not react-window for simplicity, re-evaluate if performance issues)

4. **New Check-in Animation:**
   - When `newCheckIn` is received via WS, briefly highlight the newly checked-in student (CSS animation, fade in green)
   - Show a brief toast/notification: "${name} 已签到"

5. **Layout:**
   - On desktop (≥md): Side-by-side layout — QR code left, dashboard right
   - On mobile (<md): Stacks vertically — QR on top, dashboard below
   - Current QR code and session controls remain unchanged
</action>

<acceptance_criteria>
- Status panel shows attendance rate as a large colored percentage with progress bar
- "已签到 X / Y 人" count updates in real time
- Present list and absent list are displayed side by side on desktop
- Each list is scrollable with a max height
- New check-in from WS triggers a brief highlight animation on the newly checked-in student
- Mobile layout stacks vertically
- Existing session controls (end button, QR code, countdown) remain functional
- TypeScript: no type errors
</acceptance_criteria>
