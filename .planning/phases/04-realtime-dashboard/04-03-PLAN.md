# 04-03: Client WebSocket Hooks

## Objective

Create a React hook that manages WebSocket connection lifecycle for the teacher's check-in page: connect, subscribe to session room, auto-reconnect with exponential backoff, and fall back to polling when WebSocket is unavailable.

## Tasks

### Task 1: Create useCheckinSession hook

<read_first>
- src/lib/ws-types.ts (from 04-01)
- src/app/(protected)/checkin/[courseId]/components/CheckInPageClient.tsx (existing polling logic)
</read_first>

<action>
Create `src/lib/use-checkin-session.ts`:

```typescript
interface UseCheckinSessionOptions {
  sessionId: string;
  // Callbacks
  onAttendanceUpdate: (payload: AttendanceUpdatePayload) => void;
  onSessionEnded: (payload: SessionEndedPayload) => void;
  onError: (error: string) => void;
  // Connection
  wsUrl?: string; // defaults to window.location.origin with ws://
}

interface UseCheckinSessionResult {
  connectionStatus: "connecting" | "connected" | "disconnected";
  subscribe: () => void;
  unsubscribe: () => void;
}
```

Implementation:
1. Use `useRef` for WebSocket instance (persist across renders)
2. `connect()`:
   - Determine WS URL from `window.location.origin` (replace `http` with `ws`)
   - Create `new WebSocket(wsUrl)`
   - `onopen`: set status to "connected", send `{ type: "subscribe", sessionId, payload: { sessionId } }`
   - `onmessage`: parse JSON, dispatch by type to callbacks
   - `onclose`: set status to "disconnected", start reconnect timer
   - `onerror`: call onError callback
3. Auto-reconnect with exponential backoff:
   - Initial delay: 1s
   - Max delay: 30s
   - Multiplier: 2x
   - Reset on successful connection
4. `disconnect()`: close WS, clear reconnect timer
5. Cleanup on unmount: disconnect
6. Return `{ connectionStatus, subscribe, unsubscribe }`
</action>

<acceptance_criteria>
- `src/lib/use-checkin-session.ts` exists with exported hook
- Connects to WS server on mount
- Sends `subscribe` message on open
- Dispatches incoming messages to correct callbacks
- Auto-reconnects with exponential backoff on disconnect
- Cleans up on unmount
- TypeScript: no type errors
</acceptance_criteria>
