# 05-01: History API Endpoints

## Objective

Create three API endpoints and add the "历史记录" button to course cards.

## Tasks

### Task 1: GET /api/courses/[courseId]/sessions

<read_first>
- src/app/api/sessions/[sessionId]/route.ts (existing session API pattern)
- src/db/schema/checkin.ts
- src/lib/checkin-service.ts
</read_first>

<action>
Create `src/app/api/courses/[courseId]/sessions/route.ts`:

1. Teacher auth check (getAuthSession, verify ownership of the course)
2. Query all `check_in_sessions` for this course, ordered by `created_at DESC`
3. For each session, include: id, token, status, expiresAt, createdAt, closedAt, checkedInCount (subquery or separate query)
4. Return: `{ data: { sessions: [...], total: number } }`
5. Support `?limit=20&offset=0` for pagination
</action>

<acceptance_criteria>
- `curl /api/courses/{courseId}/sessions` returns all sessions for that course, newest first
- Each session includes checkedInCount
- Pagination works (limit/offset)
- Unauthorized teachers get 401
- TypeScript: no type errors
</acceptance_criteria>

### Task 2: GET /api/students/[studentId]/attendance

<read_first>
- src/db/schema/students.ts
- src/db/schema/checkin.ts (attendanceRecords table)
</read_first>

<action>
Create `src/app/api/students/[studentId]/attendance/route.ts`:

1. Teacher auth check
2. Verify the student exists and belongs to a course the teacher owns
3. Return all attendance records for this student: each record includes session info (date, status, course name)
4. Also return summary stats: totalSessions, presentCount, absenceCount, attendanceRate
5. Return: `{ data: { student: { id, studentId, name }, summary: { ... }, records: [...] } }`
</action>

<acceptance_criteria>
- Returns student info + attendance records + summary stats
- Teacher must own the course the student belongs to
- 404 if student not found or teacher doesn't own the course
- TypeScript: no type errors
</acceptance_criteria>

### Task 3: GET /api/courses/[courseId]/attendance-summary

<read_first>
- src/db/schema/students.ts
- src/db/schema/checkin.ts
</read_first>

<action>
Create `src/app/api/courses/[courseId]/attendance-summary/route.ts`:

1. Teacher auth + ownership check
2. For each student in the course, compute: totalSessions (for this course), presentCount, absenceCount, attendanceRate
3. Sort by attendanceRate ascending (worst attendance first by default)
4. Support `?sortBy=attendanceRate|studentId|name&sortOrder=asc|desc`
5. Support `?limit=50&offset=0`
6. Return: `{ data: { students: [...], total: number } }`
</action>

<acceptance_criteria>
- Returns all students with attendance stats for the course
- Default sort: worst attendance first
- Pagination works
- Teacher ownership enforced
- TypeScript: no type errors
</acceptance_criteria>
