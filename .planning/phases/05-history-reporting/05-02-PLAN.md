# 05-02: Session History Pages

## Objective

Create the session history list page and session detail page for the teacher.

## Tasks

### Task 1: Session history list page

<read_first>
- src/app/(protected)/dashboard/page.tsx (existing dashboard pattern)
- src/app/(protected)/dashboard/components/course-card.tsx (card pattern)
- src/app/api/courses/[courseId]/sessions/route.ts (from 05-01)
</read_first>

<action>
Create `src/app/(protected)/dashboard/courses/[courseId]/history/page.tsx`:

1. "use client" component
2. Fetch sessions from `/api/courses/[courseId]/sessions` on mount
3. Display:
   - Header with course name and back link to dashboard
   - List of past sessions as cards (similar style to course cards)
   - Each card shows: date (format: "2026年6月1日 10:30"), status (已完成/已过期), attendance rate with colored badge, checkedInCount/totalStudents
   - Click card → navigate to session detail page (`[sessionId]`)
4. Empty state: "暂无签到记录" when no sessions exist
5. Loading state: spinner
6. Error state: error message with retry button
</action>

<acceptance_criteria>
- Page loads list of past sessions for the course
- Each session card shows date, status, attendance rate
- Click session card navigates to detail page
- Empty state shows "暂无签到记录"
- Loading state shows spinner
- Error state shows error + retry
- Back link returns to dashboard
- TypeScript: no type errors
</acceptance_criteria>

<action>
Create `src/app/(protected)/dashboard/courses/[courseId]/history/[sessionId]/page.tsx`:

1. "use client" component
2. Fetch session details from `/api/sessions/[sessionId]` on mount
3. Display:
   - Header: course name, back link to history list
   - Session info card: date/time, duration (createdAt to closedAt/expiresAt), status badge (已完成/已过期)

     - 未签到 (gray): scrollable table with student ID + name
4. Loading state: spinner
5. Error state: error message with retry
</action>

<acceptance_criteria>
- Shows session info: date, duration, status

- Shows present and absent student lists
- Back link returns to history list
- Loading/error states handled
- TypeScript: no type errors
</acceptance_criteria>
