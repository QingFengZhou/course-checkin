  <behavior>
    - POST /api/sessions/[sessionId]/manual-checkin: marks a specific student as present
    - Auth gate: getAuthSession, return 401 if not authenticated
    - Verify session exists, is active, and teacher owns the session's course
    - Accept body: { studentId } (the student's database UUID)
    - Call submitAttendance(sessionId, studentId) from checkin-service
    - If duplicate: return 200 with { data: { alreadyCheckedIn: true } }
    - On success: return 201 with { data: { studentName, checkedAt } }
    - Student roster: add "签到" button next to each student in the table
    - "签到" button calls the manual check-in API for the CURRENT active session
    - Need to track active session in student-roster context (passed from parent or fetched)
  </behavior>
  <action>
    Create src/app/api/sessions/[sessionId]/manual-checkin/route.ts.

    Import pattern:
    ```
    import { NextRequest, NextResponse } from "next/server";
    import { db } from "@/db";
    import { courses, checkInSessions } from "@/db/schema";
    import { eq, and } from "drizzle-orm";
    import { getAuthSession } from "@/lib/auth";
    import { submitAttendance } from "@/lib/checkin-service";
    ```

    POST handler logic:
    1. Auth gate: getAuthSession(request) — 401 if not authenticated
    2. Extract sessionId from params (await params).sessionId
    3. Query checkInSessions WHERE id=sessionId, JOIN with courses to verify teacherId === session.user.id
    4. If session not found or not owned by teacher — 403
    5. If session.status !== 'active' or expired — 400 "Session is not active"
    6. Parse body: { studentId: string } (student's DB UUID)
    7. Call submitAttendance(sessionId, studentId)
    8. If duplicate — return 200 with { data: { alreadyCheckedIn: true } }
    9. On success — return 201 with { data: { success: true } }

    Modify src/app/(protected)/dashboard/components/student-roster.tsx (per D-3.08):
    - Add a "签到" button in each student row, to the left of the "移除" button
    - The button needs the activeSessionId — add it as an optional prop to StudentRoster
    - If activeSessionId is provided: clicking "签到" calls POST /api/sessions/[sessionId]/manual-checkin
    - On success: re-fetch student list (or update local state to show check-in status)
    - If no activeSessionId prop: button is disabled with tooltip "当前无进行中的签到"
    - Style: green outline button matching the existing button patterns

    Dashboard page integration: the dashboard needs to pass activeSessionId to StudentRoster when a session is active. Since the check-in page is a separate route (/checkin/[courseId]), the dashboard doesn't directly know about active sessions. Simplify: the StudentRoster component fetches active session on mount via a lightweight GET /api/courses/[courseId]/active-session endpoint. This keeps the dashboard unaware of session state.

    Alternatively, for v1 simplicity: the "签到" button in student-roster is always visible but only functional when there's an active session. The component can try to fetch the active session lazily when the user clicks "签到". If no active session exists, show a toast/message "当前无进行中的签到".
