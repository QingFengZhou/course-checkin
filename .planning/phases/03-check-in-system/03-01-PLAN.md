<task type="auto">
  <name>Task 3: Create check-in session service (createSession, getActiveSession, submitAttendance, endSession, isSessionExpired)</name>
  <files>src/lib/checkin-service.ts</files>
  <behavior>
    - createSession(courseId, teacherId): inserts check_in_sessions row with token (nanoid), status='active', expires_at=now+5min (per D-3.01). Returns session with token
    - getActiveSession(token): selects session by token where status='active'. Returns null if not found or expired. Per D-3.04, no background jobs — expiry checked at API level
    - submitAttendance(sessionId, studentId): inserts attendance_records row. Returns true on success. DB unique constraint prevents duplicates (idempotency)
    - endSession(sessionId): updates status to 'closed', sets closed_at=now
    - isSessionExpired(session): returns true if expires_at < now or status !== 'active'
    - All functions use db from @/db import (same as API route patterns in Phase 2)
    - No console.log statements (per TypeScript coding-style rules)
  </behavior>
  <action>
    Create src/lib/checkin-service.ts with five exported async functions:

    1. createSession(courseId: string, teacherId: string):
       - Generate token via nanoid()
       - Calculate expiresAt = new Date(Date.now() + 5 * 60 * 1000) per D-3.01
       - Insert into check_in_sessions: { courseId, token, status: 'active', expiresAt }
       - Return the created session (with token for QR generation)

    2. getActiveSession(token: string):
       - Select from check_in_sessions where token matches and status = 'active'
       - Return null if not found
       - Check isSessionExpired on result — if expired, call endSession and return null
       - This implements D-3.01: timeout checked at read time, no background jobs

    3. submitAttendance(sessionId: string, studentId: string):
       - Insert into attendance_records: { sessionId, studentId }
       - checked_at defaults to now() in schema
       - Return { success: true } on insert
       - If unique constraint violation (duplicate), return { error: 'already checked in' } — idempotency

    4. endSession(sessionId: string):
       - Update check_in_sessions set status='closed', closed_at=now() where id=sessionId

    5. isSessionExpired(session: SelectCheckInSession): boolean:
       - Return session.expiresAt < new Date() || session.status !== 'active'

    Use existing import patterns: `import { db } from "@/db"` and `import { eq } from "drizzle-orm"`.

    Note: nanoid must be installed (check package.json). If missing, the executor should install it per CLAUDE.md tech stack.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| student browser -> public API (/api/checkin/submit) | Untrusted input (studentId, name) crosses without auth |
| teacher browser -> protected API (/api/sessions/*) | Authenticated but input must be validated (courseId ownership) |
| API -> database | Session token used in WHERE clause — no SQL injection via Drizzle parameterization |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | check_in_sessions.token | mitigate | nanoid generates 21-char random tokens (per D-3.02), making session tokens unguessable |
| T-03-02 | Tampering | attendance_records insertion | mitigate | DB uniqueIndex on (session_id, student_id) enforces idempotency; duplicate check-ins rejected at DB level |
| T-03-03 | Information Disclosure | getActiveSession | mitigate | Session lookup by token only — no enumeration endpoint for active sessions |
| T-03-04 | Denial of Service | submitAttendance API | accept | No rate limiting in v1; 300 concurrent writes handled by PostgreSQL MVCC (per CLAUDE.md architecture decision) |
| T-03-05 | Authorization Bypass | createSession | mitigate | Teacher ownership verified: API route checks teacherId from JWT matches course.teacherId before calling createSession |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | slopcheck + blocking human checkpoint for [ASSUMED]/[SUS] |
